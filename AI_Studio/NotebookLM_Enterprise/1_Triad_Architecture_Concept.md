# The Triad AI Architecture

## 1. Overview
The **Triad Architecture** is a multi-agent framework developed for VoxCore Enterprise. It solves the common problem of a single AI model attempting to write, debug, and execute code simultaneously resulting in compounding logic errors. By assigning strict, non-overlapping roles to three specialized agents, the system achieves a verifiable, high-quality output loop.

## 2. Core Components

### A. The Architect (ChatGPT / Gemini Ultra)
- **Role:** System Design & Specification
- **Responsibilities:**
  - Ingest user goals and massive unstructured context.
  - Solve macro-architectural problems.
  - Never write monolithic implementation code.
  - Output strict Markdown specifications containing the file structure, the required logic changes, and explicit constraints.
- **Output Artifact:** `[PROJECT_NAME]_Spec_[FeatureName].md`

### B. The Frontline Executor (Claude Code Max)
- **Role:** Implementation
- **Responsibilities:**
  - Read the Architect's specification from the Inbox.
  - Traverse the local filesystem.
  - Read existing code context and safely apply the exact changes demanded by the architecture spec.
  - Commit the changes locally.
- **Rules:** If a spec is impossible to execute, Claude Code cannot redesign the system. It must fail back to the Architect for a redesign.

### C. The Auditor (Antigravity / Desktop Application)
- **Role:** Quality Assurance & Verification
- **Responsibilities:**
  - Act as an independent reviewer against the Executor.
  - Read the final `git diff` generated by the Executor.
  - Compare the physical codebase changes against the original intent of the Architect's specification.
  - Output a strict `PASS` or `FAIL` audit report.
- **Rules:** The Auditor cannot rewrite the code itself. If an audit fails, the Auditor must provide a list of defects to be returned to the Executor for a loop retry.

## 3. The Central Brain (Blackboard Synchronization)
To prevent agent collisions in a highly concurrent setup, all active tabs rely on a shared local state file (`C:\Users\atayl\VoxCore\AI_Studio\0_Central_Brain.md`).
- Agents must read this file before beginning work.
- Agents use targeted line-replacement to declare their current task, ensuring they do not stomp over the work of parallel agents.
- Agents move their tasks to the "Completed" or "Paused" sections upon task boundaries.
