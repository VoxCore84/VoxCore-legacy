What is MCP? Concept

Model Context Protocol — a standard for giving AI tools access to external data. Each MCP server adds new tools Claude can call.

Think of it as: Plugins that give Claude superpowers beyond reading files.

Config: ~/.claude/settings.json under "mcpServers"

MCP You use this

Model Context Protocol — open standard (Anthropic) for AI ↔ tool connections. Like USB-C for AI.

  • You have: MySQL, GitHub, Wago DB2, Code Intelligence
  • How: Server runs as subprocess, Claude calls its tools natively

Agentic AI

AI that takes actions autonomously: observe → think → act → observe → repeat.

  • vs Chatbot: Chatbot gives answers. Agent does work.
  • Your example: /build-loop builds, reads errors, fixes, rebuilds

RAG

Retrieval-Augmented Generation — retrieve relevant info at query time instead of relying on training data.

  • Wago lookups → CSV data → Claude reasons
  • MCP MySQL → live DB query → grounded answers
  • Glob/Grep → find code → edit with context

A2A Industry

Agent-to-Agent Protocol (Google) — agents from different vendors communicate. MCP = AI-to-tool, A2A = AI-to-AI.

ACP Industry

Agent Communication Protocol — structured message passing between agents with capability advertisement.

ANP Industry

Agent Network Protocol — agent discovery and networking. Like DNS for agents.

Context Window

Claude's finite "working memory". Your setup manages it via:

  • Auto-compact at 80% — older messages summarized
  • Sub-agents — each gets own context
  • MEMORY.md — persistent across sessions
  • /compact — manual compress

Tool Use

Core mechanism: AI outputs structured tool call → system executes → returns result.

  • Built-in: Read, Edit, Write, Bash, Glob, Grep
  • MCP: mysql, github, wago-db2, codeintel
  • Permission modes: auto-allow or require approval
